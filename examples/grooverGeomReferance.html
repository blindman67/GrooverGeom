Groover.Geom Referance

Groover is a broader scope and includes many private (not for public) and general objects. Geom can be easily moved as current it is not dependent on anything else in the Groover name space. Though this may change in the future.

Geom is very much in the development process and is a private enterprise by Mark Spronck. It is in this current form free to do with what you wish but will move to a more detail licence when considered ready for use. Groover and all its types is currently copywrite Mark Spronck 2014-2016 (with the exception Groover.Geom) and should not be copied in any way without the express writen permission of Mark Spronck.

As you can see from this page Geom is a rather extensive object, and due to the nature of the development process (part time in my bedroom) it is subject to constant refinment, deletions, and refactoring. There is a small level of legacy support as changes move the project on but these are ad-hock and only remain while testing apps requier the support

Geom is designed to be a set of utilities and Objects for working with 2D geometry for the JavaScript environment. I started to develop this as I had a constatant need for a veriaty of geometry based functions, and thus had gathered a rather disorganised collection of functions that where incompatable with each other and inconsitant in functionality. Geom is the result of a consurted effort to normalise this collection and bring consistance to 2D geometry within the Groover namespace.

Its primary design requierment is speed, thus... Geom is not, nor will it ever be an OO design. Geom is not, nor will it ever be cosidered Functional (programming)

With rare exception null will not be used, tested for, or returned.It will only ever be used when the term null has sematic and cotextual meaning and that meaning is inconsitant with undefined.

All equalities will always use === or !==. If you see == or != it is a mistake (late night development) and I will fix it as soon as I become aware of the mistake. I do not at any point test if a Javascript basic type is of the correct type and thus it is possible for string to creep into the primitives, this can result in ambiguity. If you are unaware of the differance between === and == or how type coercion is implemented in Javascript then Geom will not be of any benifit to you, as it will likely result in frustration.

List of types

  • Primitives

    I am trying to keep this as simple as posible though there may at any point come more primitives if the need is deamed worth the effort.

    A primitive can become empty. When empty the primitive can not be relied upon to represent anything meaning full. Every effort is being made to have empty primitive not throw any errors but the result of any function that uses an empty primitive as an argument, or is its self empty and returning a primitive will also result in more empty primitives. The exact nature of the empty state is dependent on the type. An empty primitive will always returns true to the function call Geom.isEmpty or [abstract primitive].isEmpty() (where [abstract primitive] is any primitive type.). Though it is possible for the function Geom.isEmpty(object) to return undefined. This means that it could not find the requiered propeties and functions needed to determin the state of the primitive. In such a case the object should not be considered as part of the Geom name space.

    All pprimitives share some common properties and functions

    • Common Properties

      • type

        The primitives type as a string eg Geom.Vec has a type === "Vec" you will be able to use type to create a new object of the same type. eg var vec = new Geom[this.type](), or convert if posible one object to another. Eg prim.as[prim2.type]() will return prim as the same type as prim2

      • lableStr

        A string of what ever you like. Note that JavaScipt string are 16 bit Unicode characters. Take care when serilising this property for transport or storage.

      • id

        A number representing a identifiable group, or individual item, type or catagory, or undefined. The function make unqiue changes the type to a value that is unique in the current Javascript context and unqiue only primitives that have had that function called. Primitives that have not had this function called and have an id that is not undefined will corrupt the term unique within the current context
